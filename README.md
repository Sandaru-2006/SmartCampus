# Smart Campus Sensor & Room Management API

## 1. API Overview
The Smart Campus API is a backend system designed to simulate real-world facilities management and smart building automation. It provides a RESTful interface to manage physical university campus rooms, the smart sensors installed within them, and the historical readings generated by these sensors.

## 2. Build Steps
Ensure Java (JDK 21) and Maven are installed on your system.
1. Open a terminal in the project root directory.
2. Compile and package the application:
   ```bash
   mvn clean compile
   ```

## 3. Run Steps
1. Start the server using Maven:
   ```bash
   mvn exec:java "-Dexec.mainClass=Main"
   ```
2. The server will start and listen for requests at `http://localhost:8080/api/v1`.

## 4. cURL Command Examples

**1. Discover the API:**
```bash
curl -X GET http://localhost:8080/api/v1
```

**2. Create a Room:**
```bash
curl -X POST http://localhost:8080/api/v1/rooms \
  -H "Content-Type: application/json" \
  -d "{\"id\":\"R101\", \"name\":\"Main Lecture Hall\", \"capacity\":150}"
```

**3. Create a Sensor (Assuming Room R101 exists):**
```bash
curl -X POST http://localhost:8080/api/v1/sensors \
  -H "Content-Type: application/json" \
  -d "{\"id\":\"S1\", \"type\":\"CO2\", \"status\":\"ACTIVE\", \"roomId\":\"R101\"}"
```

**4. Filter Sensors by Type:**
```bash
curl -X GET "http://localhost:8080/api/v1/sensors?type=CO2"
```

**5. Add a Sensor Reading:**
```bash
curl -X POST http://localhost:8080/api/v1/sensors/S1/readings \
  -H "Content-Type: application/json" \
  -d "{\"id\":\"RND1\", \"timestamp\":1713513600000, \"value\":410.5}"
```
